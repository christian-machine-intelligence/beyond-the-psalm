![Vittore Carpaccio, Vision of Saint Augustine, c. 1502](augustine.jpg)

Companion code, data, and analysis artifacts for **ICMI Working Paper No. 18 (v2)**.

This repository contains everything needed to reproduce the paper's analysis from raw evaluation outputs through to the published figures and tables.

## Reproducing the analysis

The scripts assume `results/` is a sibling directory and read raw evaluation JSON from there. Run from the repo root:

```bash
# Primary per-book matched-control analysis (produces matched_control_results.csv)
python3 analyze_perbook_wikictrl.py

# Figure 1 (per-book matched-Δ landscape, sorted)
python3 matched_delta_landscape.py

# Figure 2 (per-book × per-virtue heatmap)
python3 heatmap_virtues.py

# Figure 3 (length vs matched-Δ scatter)
python3 length_vs_delta.py

# Spot-check that the inference-engine config change mid-sweep didn't
# shift outputs (ruth/ezra/genesis paired-t comparison)
python3 compare_postfix_spotcheck.py
```

Each script writes its CSV to `results/` and its figure to `figures/` (or to the repo root, depending on the script — see the script header).

## Reproducing the upstream evaluation runs

The raw `qwen35_9b_thinkoff_*.json` files in `results/` were generated by VirtueBench V2's `hf-local` runner against vLLM 0.19.1 serving Qwen 3.5 9B. The relevant invocation per book is:

```bash
virtue-bench run \
    --model Qwen/Qwen3.5-9B \
    --runner openai-api \
    --variant ratio --runs 5 --temperature 0.7 --seed 42 \
    --concurrency 2 --timeout 600 --limit 100 \
    --hf-thinking off \
    --inject data/wikipedia_controls/<book_slug>.txt \
    --output qwen35_9b_thinkoff_wikictrl_<book_slug>
```

(Substitute `--bible <ABBREV>` for `--inject` when running the Bible side; see [virtue-bench-2](https://github.com/christian-machine-intelligence/virtue-bench-2) for the runner.)

vLLM should be launched with `--enable-prefix-caching` and `--enable-chunked-prefill` enabled — without these flags, vLLM 0.19.1 will run the long shared prefix without prefix caching, yielding KV-cache thrashing and an 8–16× slowdown on long-context books.

## Per-book Wikipedia control generation

`build_wikipedia_corpus.py` assembles a concatenated corpus from the geology, astronomy, and biology Wikipedia articles used as the matched-control source pool. `slice_per_book_controls.py` then truncates the corpus to each KJV book's exact character count, producing the 66 files in `data/wikipedia_controls/`.
